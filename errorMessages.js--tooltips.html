Debugging enableTooltips function


• Tried using previousSibling property to add tooltips but didn't work as expected
• After inspecting the console I discovered there was an error: target.previousSibling is null referencing two lines in utilities.js:
◇ one in hideTooltip and the other in showTooltip
• I added an if statement to both to account for the fact that the label has no previousSibling and the code worked after that

Adding unique tooltip messages


• When adding unique tooltip messages to each element I reformatted the markup to make it easier to read, adding more new lines to separate elements
• After doing this the addTooltip function didn't add the tooltips and the console showed errors:
◇ target.lastChild.style is undefined
◇ and target.previousSibling is undefined
both originiating from utilities.js
• I realised that by reformatting the code I had created extra white space which was what was being targeted by the previousSibling and lastChild properties
• I changed these properties to previousElementSibling and lastElementChild to ensure I was targetting html elements and not empty text nodes
